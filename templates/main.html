Welcome to the LP Reports application!

Here you can find various dashboards for bugs on Launchpad mostly for product (Fuel + MOS) development.

Highlights

When showing a report, the application searches simultaneously through 'Fuel' and 'MOS' Launchpad projects, making that distinction transparent.

You can find the most actual reports in the 'Release' and 'Triage' menus above. The reports located in 'Project reports' are not worked on for some time, but they still should work.

Some reports provide filtering capability allowing you to further narrow down list of bugs you need. The most interesting filter is the one helping you to see bugs assigned to the specific team. The teams are defined in this file in our repo. Note that the entities which are inside 'teams' list are actually references to launchpad teams with the corresponding names. E.g. mos-ceilometer is a reference to mos-ceilometer team on Launchpad.

Accessing Private Bugs

You need to log in if you want to see Private or Private Security bugs. If you push 'Login' button, you will be redirected to Launchpad
where you can select duration of the token you provide to the application to search private bugs for you. Note a few things:

  • Once you issue a token on the launchpad page you will have to manually return back to the application by accessing http://lp-reports.vm.mirantis.net/.
  • Once you log in, report generation will become considerably slower: while application stores public bugs in local DB, it will have to query launchpad using your token to generate each report to get private bugs. It takes up to several minutes for some reports.
  • Right now the feature works only for reports in 'Release' and 'Triage' menu.
  • If you ever get stuck with authentication, access this page to return things to normal: http://lp-reports.vm.mirantis.net/logout.

Get in Touch

Also if you have questions or suggestions or need help, feel free to reach us via lp-reports@mirantis.com mailing list.
